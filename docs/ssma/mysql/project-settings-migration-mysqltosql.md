# <a name="project-settings-migration-mysqltosql"></a>プロジェクトの設定 (移行) (MySQLToSQL)
[**プロジェクトの設定**] ダイアログボックスの [移行] ページには、Ssma が MySQL から SQL Server にデータを移行する方法をカスタマイズする設定が含まれています。  
  
[移行] ウィンドウは、[**プロジェクトの設定**] ダイアログボックスと [**既定のプロジェクトの設定**] ダイアログボックスで使用できます。  
  
-   すべての SSMA プロジェクトの設定を指定するには、[**ツール**] メニューの [**既定のプロジェクト設定**] をクリックし、設定にアクセスする [**移行ターゲットバージョン**] ドロップダウンでプロジェクトの種類を選択します。左側のウィンドウの下部にある [**全般**] をクリックし、[**移行**] をクリックします。  
  
-   現在のプロジェクトの設定を指定するには、[**ツール**] メニューの [**プロジェクトの設定**] をクリックし、左側のウィンドウの下部にある [**全般**] をクリックして、[**移行**] をクリックします。  
  
## <a name="options"></a>オプション  
  
### <a name="bulk-copy"></a>一括コピー  
  
|期間|定義|  
|--------|--------------|  
|**バッチ サイズ**|データ移行中に使用するバッチサイズを指定します。<br /><br />**既定のモード**: 1000<br /><br />**オプティミスティックモード**: 1000<br /><br />**フルモード**: 1000|  
|**CHECK 制約**|SSMA が SQL Server テーブルにデータを挿入するときに、制約をチェックするかどうかを指定します。<br /><br />**既定のモード**: False<br /><br />**オプティミスティックモード**: False<br /><br />**フルモード**: False|  
|**[トリガーを起動する]**|SSMA が SQL Server テーブルにデータを追加するときに挿入トリガーを起動するかどうかを指定します。<br /><br />**既定のモード**: False<br /><br />**オプティミスティックモード**: False<br /><br />**フルモード**: False|  
|**[ID を保持する]**|SQL Server にデータを追加するときに、SSMA が MySQL の id 値を保持するかどうかを指定します。 値を False に設定すると、id 値が変換先によって割り当てられます。<br /><br />**既定のモード**: True<br /><br />**オプティミスティックモード**: True<br /><br />**フルモード**: True|  
|**[NULL を保持する]**|SQL Server で指定されている既定値に関係なく、SQL Server にデータを追加するときに SSMA がソースデータ内の null 値を保持するかどうかを指定します。<br /><br />**既定のモード**: True<br /><br />**オプティミスティックモード**: True<br /><br />**フルモード**: True|  
|**[テーブル ロック]**|データ移行中にテーブルにデータを追加するときに SSMA がテーブルをロックするかどうかを指定します。 一括コピー操作中に、一括更新ロックを取得します。 値が False の場合は、行レベルでロックが設定されます。<br /><br />**既定のモード**: False<br /><br />**オプティミスティックモード**: False<br /><br />**フルモード**: False|

|**負の時刻値の移行**|TIME 列に "-30:11:00" のような負の値を移行する方法を指定します。<br /><br />**既定のモード**: NULL の設定<br /><br />**オプティミスティックモード**: NULL の設定<br /><br />**フルモード**: NULL の設定|  
|**24時間の移行時の時間の値**|Time 列で ' 23:59:59 ' を超える時刻値を移行する方法を指定します。<br /><br />**既定のモード**: NULL の設定<br /><br />**オプティミスティックモード**: NULL の設定<br /><br />**フルモード**: NULL の設定|  
|**列に適合するようにバイナリ値を切り捨てます**|Yes の場合、SSMA では、SQL テーブルの列に合わない MySQL からバイナリ値が切り捨てられ、適切なエラーメッセージが生成されます。 ない場合は、行によってエラーが発生します。<br /><br />**既定のモード**: いいえ<br /><br />**オプティミスティックモード**: いいえ<br /><br />**フルモード**: いいえ|  
|**列に合わせる文字の値を切り捨てます**|SSMA では、SQL テーブルの列に合わない MySQL から文字値が切り捨てられ、適切なエラーメッセージが生成されます。<br /><br />**既定のモード**: いいえ<br /><br />**オプティミスティックモード**: いいえ<br /><br />**フルモード**: いいえ|  
|**日付のゼロ移行**|DATE 列と DATETIME 列で、' 0000-00-00 ' や ' 0000-00-00 00:00:00 ' のようなゼロの日付を移行する方法を指定します。<br /><br />**既定のモード**: NULL の設定<br /><br />**オプティミスティックモード**: NULL の設定<br /><br />**フルモード**: NULL の設定|  
|**ゼロの移行日**|日付と時刻の列に ' 2009-01-00 ' や ' 2000-00-00 11:00:00 ' のようなゼロ部分の日付を移行する方法を指定します。<br /><br />**既定のモード**: NULL の設定<br /><br />**オプティミスティックモード**: NULL の設定<br /><br />**フルモード**: NULL の設定|  
  
### <a name="migration-engine"></a>移行エンジン  
  
|期間|定義|  
|--------|--------------|  
|**移行エンジン**|データ移行中に使用されるデータベースエンジンを指定します。 クライアント側のデータ移行とは、SSMA クライアントがソースからデータを取得し、そのデータを SQL Server に一括挿入することを指します。 サーバー側のデータ移行とは、[SQL Server] ボックスで実行されている SSMA データ移行エンジン (一括コピープログラム) を指します。これは、ソースからデータを取得し、SQL Server に直接挿入することによって、クライアントホップを追加する (パフォーマンス向上) ことを回避する SQL エージェントジョブとして機能します。<br /><br />**既定のモード**: クライアント側のデータ移行エンジン<br /><br />**オプティミスティックモード**: クライアント側のデータ移行エンジン<br /><br />**フルモード**: クライアント側のデータ移行エンジン|  
  
> [!IMPORTANT]  
> [**移行エンジン**] オプションが [**サーバー側のデータ移行エンジン**] に設定されている場合、新しいプロジェクト設定オプションの [ **32 ビットサーバー側のデータ移行エンジンを使用**する] が表示されます。 データの移行に32ビットまたは64ビットの一括コピープログラム (BCP) ユーティリティを使用するかどうかを指定します。

|期間|定義|

### <a name="parallel-data-migration"></a>並列データ移行  
  
|期間|定義|  
|--------|--------------|  
|**並列データ移行モード**|並列データ移行を有効にするためにスレッドをフォークするために使用されるモードを指定します。 Auto モードでは、SSMA はデータを移行するためにフォークされたスレッドの数 (既定では 10) を選択します。 カスタムモードでは、ユーザーは、データを移行するためにフォークされるスレッドの数を指定できます (最小値は1、最大値は 100)。 現時点では、並列データ移行をサポートしているのは、クライアント側のデータ移行エンジンだけです。<br /><br />**既定のモード**: Auto<br /><br />**オプティミスティックモード**: 自動<br /><br />**フルモード**: 自動|  
  
> [!IMPORTANT]  
> [**並列データ移行モード**] オプションを [**カスタム**] に設定すると、新しいプロジェクト設定オプションの [**スレッド数**] が表示されます。 データ移行に使用するスレッドの数を指定します。  
  
### <a name="spatial-data"></a>空間データ  
  
|期間|定義|  
|--------|--------------|  
|**エラーの処理**|空間データ型の値の移行に関するエラーを処理する方法を指定します。 ' Replace with NULL ' が指定されている場合、エラーの原因となったすべての空間値が NULL に置き換えられます。 それ以外の場合は、置換は行われません。<br /><br />**既定のモード**: エラーを生成します。<br /><br />**オプティミスティックモード**: エラーを生成する<br /><br />**フルモード**: エラーを生成します|  
|**値の検証**|無効な空間値を処理する方法を指定します。 ' Try Make Valid ' が指定されている場合は、無効な値を変更して有効にしようとしています。<br /><br />**既定のモード**: 有効にする<br /><br />**オプティミスティックモード**: 変更しない<br /><br />**フルモード**: 有効にします|
